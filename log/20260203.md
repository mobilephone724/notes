## question about batch tokenizer

From the python3 "transformers" library, I see that "tokenizer" can tokenize multiple sentences in one function call. But in a general LLM product, will it split the user's input into many sentences and tokenize them seperately?



1. The batch API means all input sentences are **isolate**, they are tokenized **independently**
2. In LLM product, my long input is **NOT** seperated into multiple sentences for tokenizer
3. But leverage the ability of GPU, my request is **grouped with others** and they are treated as different setences to the tokenizer and model later.
